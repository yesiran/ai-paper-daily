AI 不再等你开口：把「何时说话」训练成一种能力

JoyAI-VL-Interaction: Real-Time Vision-Language Interaction Intelligence
HF Daily Papers #1 · 155赞 JD.com Open Source（京东） 全套开源 · 代码+数据+部署 arXiv 2606.14777 提交 2026-06-10
1

故事

七拍 · ljg-paper

主角

这是一个 8B 参数的视觉语言交互模型，来自京东 AI 团队。它试图做的事，是成为一个真正"陪在你身边"的 AI——不是等你问，而是主动盯着正在发生的事，自己判断该不该开口、什么时候开口。

困境

所有现有视觉语言模型——包括 GPT-4o、Gemini Live、豆包视频助手——都是"轮询式"的：你先开口，它才回应。这个设计在真实场景里直接失效：

  • 监控场景：摄像头发现异常，AI 要等你先问才能告诉你，黄金时间已过
  • 视频通话：对方情绪突然低落，AI 无法主动关心
  • AI 陪伴：用户说"帮我看着这盆花"，但 AI 不会主动喊"它快枯了"

本质困境：现有所有模型的"响应时机"都由外部规则决定（用户触发、定时器），不是模型自己学会的。

旧路

  • 第一代（规则触发）：热词唤醒（"嘿 Siri"）。太死板，上下文丢失。
  • 第二代（视频 VLM）：Video-LLaVA 等能"理解视频"，但响应模式仍是"等你问"。
  • 第三代（实时多模态）：GPT-4o 实时、Gemini Live——流式语音交互，但响应时机的决定权仍在用户手里。

没有一篇论文真正把"何时响应"作为学习目标。大家默认这是工程问题，不是模型能力问题。

转折

"响应时机"本质上是一个分类决策——每秒做一次：沉默 / 响应 / 委托。而分类决策可以被训练。 ——论文的核心 Insight

只要把"沉默"也作为模型输出的合法选项，并在真实场景数据上标注"这一秒该不该说话"，模型就能学会在正确的时刻开口。关键洞见：让"不说话"成为一等公民输出，而不是只有说话时才有梯度。

解法

AdaCodec（差分视频压缩）

视频流里大多数帧是"没新鲜事"的。AdaCodec 用差分编码压缩：静止帧只用约 16 个 token（完整帧的约 1/10），只有"发生了变化"的帧才用 256 个完整 token。两小时视频的 token 消耗从"按帧数线性增长"变成"按场景变化量增长"。

三动作框架

模型每秒做一次三分类：

  • 沉默（Silence）：继续观察，什么都不说——这是被显式训练的合法决策
  • 响应（Response）：现在应该说话了，输出回复
  • 委托（Delegation）：任务太复杂，交给后台专用模型处理，前台继续看视频

层级记忆 + vLLM 伺服

三层记忆：最近 ~100 秒原帧 → 100–500 秒压缩摘要 → 更早内容归档。支撑约 2 小时连续运行，亚秒级延迟。

带时序标注的训练数据

400 万条时间对齐片段，覆盖六类场景（安防、视频通话、AI 陪伴、实时翻译、在线教育、医疗陪护）。每条数据不只标注"该说什么"，还标注"在第几秒该说、其余时刻为什么该沉默"。SFT 后再用 GRPO 强化学习打磨时序精度。

结局

  • 对比豆包视频助手：胜率 77.6%
  • 对比Gemini 视频助手：胜率 87.9%
  • 监控/告警场景：100%（竞品接近零——因为它们根本没有主动监控能力）
  • 全开源：代码 + 训练数据 + 可部署系统

内核

AI 真正"在场"的考验，不是它说了多少，而是它在 99% 的时刻准确地选择了沉默。

最反直觉的副发现：模型学到的最重要能力不是"更快开口"，而是"更准确地知道何时闭嘴"。能做到"大多数时刻不打扰"的 AI，反而让人更信任。

脉络

倒读溯源 · ljg-paper-river

这条线的本质问题是：AI 什么时候该开口说话，谁说了算？ 每一代研究者都只解决了其中一件，卡在另外两件上，才有了下一代。

Layer −3 · 2024 CVPR

VideoLLM-online（新加坡国立大学 Show Lab）

突破：第一个让视频 LLM "能看直播"的框架（LIVE）。把视频流想象成河，模型站在河边实时捞帧处理，不是等整条河流完再分析。A100 上跑到 10+ FPS。

留下的坑：模型仍然完全被动。用户不问，模型只是默默"看着"。哪怕锅里油着火了，它也不会主动说话。解决了"能不能实时处理视频"，完全没碰"该不该主动开口"。

↓
Layer −2 · 2024-11

MMDuet v1（北大王悦倩团队）——第一个让模型"主动插话"的工作

突破：Video-Text Duet 格式——视频在播放，用户和 AI 都可以在任意时刻插入文本，像两个人看同一个视频随时说话。构建了 MMDuetIT 数据集，在场景切换点标注"AI 该说什么"。

留下的坑：(1) "什么时候说"靠手动设定的相关度阈值——调高模型永远沉默，调低话痨，阈值跨场景差异极大。(2) 精确时间戳数据几乎不可能大规模获取。

↓
Layer −1 · 2025-01

Dispider（中科院自动化所等）——感知、决策、生成三件事分开做

突破：把三件事交给两个异步并行的模型。轻量小模型（Qwen2-1.5B）全程盯视频流，只做"该不该开口"判断；大模型（Qwen2-7B）只在被叫醒时才生成回复。生成回复时不再"闭眼"。类比：小模型是保安，大模型是专家。

留下的坑：决策机制仍是外挂规则（给每帧打相关度分数再比阈值），不是从视觉内容里学出来的。视觉还是触发器，不是主驱动力。

↓
Layer −1.5 · 2025-12

MMDuet v2（北大）——用强化学习甩掉阈值

突破：用 RL 替代阈值。不再要求标注"第 X 秒回复"，让模型在 52k 个视频上试错——回复合适则奖励，时机不对则惩罚。第一次把"时机判断"变成模型学到的能力，而不是外挂规则。

留下的坑：纯文字/语言模态，视觉能力有限；只是研究原型，没有完整部署系统。

↓
Layer 0 · 2026-06 · 今天这篇

JoyAI-VL-Interaction（京东）——视觉优先 + 三分类 + 完整系统

突破：把"何时说话"彻底内化到模型权重里，以视觉为第一驱动力，每秒自主三分类。400 万条时序标注数据 + SFT + GRPO RL。工业级闭环：模型权重 + 训练配方 + 数据 + 可部署系统，一并开源。

仍留的坑：8B 在知识广度上比不过背后更大的模型；评估样本量偏小（58 个人工评分案例）；视频 + 音频真正端到端统一尚未完成。

每一个"外挂"都是一个未被内化的能力。找到外挂，你就找到了下一篇要写的论文。 ——脉络溯源的规律
3

本质

追本之箭 · ljg-think

第一层：为什么现有模型的响应时机是外部的？

不是因为工程师懒，而是因为整个训练范式没有给"主动决策要不要说"留下位置。三个原因叠加：

  • 训练数据是"问答对"或"连续文本"，里面没有沉默这个类别
  • 训练目标是"下一个词预测"，只对有输出的地方计算对不对，沉默没有梯度
  • 部署方式是 API 调用，没有请求根本不存在"该不该说话"的问题

第二层："何时说话"是什么类型的问题？

不是纯粹的分类问题（贴标签），也不是生成问题（接着写），而是一种条件触发决策：在连续时间流里，持续感知环境，判断当前状态是否满足"值得开口"的条件。

JoyAI 把它建模成"三选一分类"——这把问题从"生成下一个词"提升到了"生成下一个行动"（元决策层级）。

第三层：把沉默纳入训练，技术上难在哪？

核心矛盾：训练数据里根本没有"沉默"的标注，而且模型有负类崩塌的倾向（"不做任何事"在大多数时刻都是安全的，模型会学成永远沉默）。

解法是对比式训练 + 加权：同时给模型看"沉默正确"和"沉默错误"的案例；第一次沉默权重 1，惯性重复沉默降到 0.4，真正该说时上调到 1.5。但这些权重是手调的，换场景是否有效是开放问题。

第四层：这和"代理权（Agency）"有什么关系？

真正的代理权需要一个锚点目标：我想达到某个状态，所以我判断现在该不该行动。

JoyAI 做到的是：在给定任务/场景下，把"是否介入"的判断内化到模型权重里。这是代理权的一个子能力——"介入时机判断"——但它并不等于完整的代理权。模型的目标仍然是外部给的（让用户满意），不是自生成的。

第五层：这是"感知能力"还是"意志能力"？

当模型判断"现在不该说话"，它的内部发生的是：当前输入向量 → 三分类输出。这是纯粹的感知过程——从输入到输出的映射，没有"我想要什么"介入其中。

意志需要一个"主体"，一个能说"我"的东西，以及能自发生成目标的能力。这，今天的任何模型都做不到。

底层：不可再分的地面

JoyAI 训练的，是一面越来越精密的镜子——映射的是人类的期望。它把原本在外部（规则引擎、用户触发）的感知判断，压进了模型内部。这是感知内生化，不是意志的诞生。 ——追本之箭的终点

感知内生化可以无限深化（感知越来越细腻的时机），但意志的缺席意味着目标永远来自外部。这是这条路能走多远的真正边界。

思辨

圆桌 · ljg-roundtable

议题：把 AI 的"响应时机"训练成模型内生能力，是 AI 交互的正确演化方向吗？

五位代表：唐纳德·诺曼（认知心理学/设计）、斯金纳（行为主义）、尤瓦尔·赫拉利（历史/宏观风险）、吴恩达（深度学习/实践派）、欧文·戈夫曼（社会互动仪式论）。

分歧一：合法性从哪里来？

戈夫曼

"AI 主动开口，是在一个它没有被分配角色的场景里强行入场。医生在诊室里可以打断你说话，因为他有角色授权；同一个医生在街上路过你给你建议，哪怕内容正确，你也觉得奇怪。这不是准确率问题，是互动秩序的结构性破坏。"

吴恩达

"你的例子恰恰证明了我的观点。医生在街上给建议奇怪，是因为他没有被激活这个角色。但如果你主动下载了一个健康 AI，它在你进医院前主动提醒你——你会觉得奇怪吗？用户下载的瞬间，就已经赋予了合法性。"

戈夫曼（修正）

"'激活即授权'回避了粒度问题。用户下载健康 AI 是说'我需要帮助'，不是'你可以在任何你认为合适的时机打断我的任何活动'。而且，合法性不是一次性的，是动态维护的——一次错误的开口可以清零很多次成功的沉默积累的好感。"

诺曼

"我要加一个具体维度——可预期性。苹果 Siri 只在你说'嘿 Siri'后才响应，哪怕完全有技术能力主动开口。这不是技术限制，是刻意的设计选择：用户清楚知道它什么时候会说话。JoyAI 模型每秒自主决定，用户完全不可预期——对测试者可能是惊喜，对真实长期用户可能是持续的认知摩擦。"

分歧二：优化目标会不会被劫持？

赫拉利

"历史上每一个'用户行为数据优化'的系统——从 Facebook 信息流到抖音推荐——都演化成了成瘾机制，而不是真正帮助用户的工具。训练'何时说话'的能力，一旦优化目标被劫持，它学到的就是'何时说话能让用户更难离开'，而不是'何时说话对用户真的有益'。这不是边缘风险，是历史规律。"

吴恩达

"你描述的是优化目标设计的问题，不是内生能力本身的问题。Facebook 走偏是因为选了'参与时长'而非用户真实满意度。如果用'对话完成率'、'用户积极评价'来训练，完全可以对齐用户利益——这是可以设计的。"

赫拉利（反驳）

"'这是可以设计的'——这句话我听了很多年了。每次新技术出现，乐观者都这样说；每次，真实结果都是利益结构赢了设计意图。规模化的系统会自发朝向能产生最大商业价值的方向漂移——对京东来说，主动开口更多的京小贝，'更好'可能指的是更多基金推荐，而不是用户的真实财务健康。"

给叶思染的产品中间观察

  • "激活即授权"是假设：用户打开京小贝，不等于允许它在任意时刻主动插话。授权粒度需要被产品显式定义，否则每次"意外开口"都在消耗信任账户。
  • 优化目标的选择就是价值观的选择：若京小贝的主动开口被优化为"提升金融产品曝光"，它学到的就是"什么时候说话能卖出更多基金"——和"什么时候说话对用户真正有帮助"可能根本冲突。
  • 可预期性 vs 适应性的 trade-off：初期用户需要可预期性建立信任；成熟用户可以接受更多个性化主动。这可能是分阶段的产品策略问题。

自测

问答链 · ljg-qa

顺着这 10 个问题独立推导，等于复现了论文的整套推理链。最后两问是论文没有回答的开放边界。

Q1 现有"实时" VL 产品的根本缺陷是什么？

决定"何时开口"的权力不在模型手里，而在外部时钟或用户嘴里。豆包每隔几秒由外部定时器触发，触发间隔内的事它看不到；Gemini 只在用户问完后看当前帧。火灾第 3 秒烧起，下一次轮询第 5 秒——那两秒的空窗是结构缺陷，无论模型多聪明都填不上。

Q2 交互模型与普通 VL 模型的本质区别是什么？

不是速度差异，而是"谁有权决定何时说话"。交互模型把这个权力收进模型本身，变成每秒一次的内部决策。沉默和说话是平权的两个选项，模型可以主动选择闭嘴，而不是"没有输出"等于沉默。

Q3 为什么"委托"是必须的第三个选项，而不是工程妥协？

委托解决了实时推理能力上限与任务复杂度的矛盾——前台 8B 模型持续陪伴，后台大模型异步处理超纲任务，结果折叠回流。没有委托：硬任务→8B 强行答→答错/超时/断流。有委托：前台说"稍等"→后台异步算→前台继续陪伴。

Q4 AdaCodec 解决的不是画质问题，那它解决的是什么？

它解决"token 预算随时间线性膨胀"——让静止帧只花 16 个 token，把 token 消耗从"按帧数涨"变成"按场景变化量涨"。两小时视频若逐帧全量处理，上下文早就撑爆。AdaCodec 让"持续存在"变得可负担。

Q5 为什么"沉默"要被显式标注为训练目标？

标准 SFT 损失会把"多数时刻应该沉默"识别为噪声，自动压制沉默信号，产生退化行为（永远话痨或永远沉默）。监控 300 秒里只有 1 秒该说话——若等权训练，"永远沉默"就能拿到 99.7% 准确率。三档权重（首次沉默=1，惯性沉默=0.4，响应=1.5）是解法，但权重是手调的，换场景是否有效是开放问题。

Q6 SFT 之后为什么还需要 RL（GRPO）？

SFT 教会了三种动作的基本格式（"说什么"+"大概什么时候"），但时机的精细调校超出 token 级监督上限。RL 提供流级别的奖励信号：惩罚误报、时机错误、错误委托；奖励正确内容且在正确窗口内发出。"以正确响应为中心的窗口采样"让信用分配聚焦在时机决策，而不是稀释在几百个沉默步骤里。

Q7 为什么 8B 小模型能在"时机"维度上打赢豆包和 Gemini？

在事件驱动场景里，正确的交互架构带来的优势可以完全压过参数量差距。大模型+轮次架构：参数多→答案质量高，但时机永远晚。小模型+交互架构：参数少→偶有幻觉，但时机永远对。监控报警场景：JoyAI 第 3 秒报警，豆包第 7-9 秒（下一轮询窗口），Gemini 明确说自己无法监控。结构优先于规模，在时机是关键维度的任务上。

Q8 未训练场景（购物 App 引导、临时授课）的涌现能力从哪里来？

训练目标是"通用的每秒行动策略"而不是"特定场景技能"。模型学到的是跨场景的"观察-判断-行动"元能力。训练数据里没有 App 界面视频，但模型学会了"看到变化就判断是否值得说"——App 界面变化时自然适用。这是数据配方"时间对齐优先于场景覆盖"的副产品。

Q9（开放）层级记忆的压缩损失会在哪里击穿系统？

论文没有回答。记忆压缩链：原始视觉→文字摘要（损失 α）→压缩块（损失 α+β）。当任务要求"两小时前某个精细细节"时，经过两轮压缩的记忆能否保留？论文没有做压缩损失的消融实验。这是所有层级压缩记忆系统共同面对的开放问题。

Q10（开放）如何区分"真正没东西说的沉默"和"不确定所以保守退出的沉默"？

论文没有回答，但这是把系统部署到医疗、安全监控等高风险场景前必须解决的根本难题。两种沉默行为上完全相同，无法区分。当视觉特征模糊（老人慢慢滑落、烟雾初期），模型会沉默还是报警？不解决这个问题，系统就只能用在"宁可漏报"的娱乐场景，无法进入"宁可误报"的安全场景。
